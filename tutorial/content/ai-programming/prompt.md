# 提示工程（Prompt Engineering）

## 什么是提示工程？

提示工程是指设计和优化输入给AI模型的文本提示（Prompt），以获得更好、更准确的输出结果的技术。它是与大型语言模型交互的核心技能。

## 基本原则

### 1. 清晰明确
- 使用简洁、明确的语言
- 避免歧义和模糊的表达
- 提供具体的上下文信息

### 2. 结构化提示
```
角色：你是一位经验丰富的Python开发者
任务：请解释以下代码的工作原理
代码：
[代码内容]
要求：
1. 逐行解释
2. 指出潜在问题
3. 提供改进建议
```

### 3. 提供示例（Few-shot Learning）
通过提供输入-输出示例，让模型理解期望的输出格式：

**示例：**
```
输入：今天天气真好
输出：正面

输入：这部电影太无聊了
输出：负面

输入：这个餐厅的食物还可以
输出：中性

输入：[待分类文本]
输出：
```

## 常用技巧

### 角色设定
为AI设定一个特定角色，可以显著提升输出质量：

```
你是一位专业的数据科学家，擅长用通俗易懂的方式解释复杂的机器学习概念。
请解释什么是过拟合。
```

### 思维链（Chain-of-Thought）
引导AI展示思考过程：

```
问题：一个农场有鸡和兔子，共有35个头和94只脚。鸡和兔子各有多少只？

请逐步思考并解答这个问题。
```

### 输出格式控制
明确指定输出格式：

```
请总结以下文章的主要内容，并以JSON格式输出：
{
  "标题": "",
  "核心观点": ["", ""],
  "关键数据": {},
  "结论": ""
}
```

## 高级技巧

### 1. 系统提示（System Prompt）
设置全局行为准则：

```
系统：你是一个乐于助人的AI助手，回答要简洁明了，避免冗长。

用户：什么是机器学习？
```

### 2. 上下文管理
维护对话历史，提供必要的背景信息：

```
[之前的对话内容]

基于以上讨论，请继续回答：...
```

### 3. 自我一致性检查
让AI验证自己的答案：

```
请回答以下问题，然后检查你的答案是否正确：
[问题]

检查步骤：
1. 重新理解问题
2. 验证计算过程
3. 确认最终答案
```

## 常见错误

1. **提示过于模糊**："写点什么"
2. **缺乏上下文**：没有提供必要的背景信息
3. **期望过高**：要求模型完成超出能力范围的任务
4. **忽视安全**：未设置适当的内容过滤

## 最佳实践

- 迭代优化：根据输出结果不断调整提示
- A/B测试：比较不同提示的效果
- 记录有效提示：建立提示模板库
- 考虑模型特性：不同模型对提示的敏感度不同

> "好的提示就像好的编程，需要清晰的逻辑、恰当的抽象和持续的优化。"
